Verified Self-Improvement · v0

A self-improving agent
must not trust its own changes.

VSI-0 is a self-verifying agent: it treats every edit to its own {prompt, rule, memory, tool} as untrusted, and adopts it only if it clears a hidden test (it transfers to held-out cases) and a regression check (it breaks no existing capability).

The field's problem: self-improvement measured on what it learned from is reward-hackable — UC-Berkeley (Apr 2026) broke all 8 major agent benchmarks to ~100%. Without a trustworthy verifier, a self-improvement loop converges to a confident reward-hacker. VSI-0 builds that verifier and stress-tests it — in text, in real code, and in an embodied driving domain, on one RTX 5090.

Flagship — safe self-modification, end to end

An agent that already handles task A learns task B by editing a rule. It rejects even the edit that fixes B best — because that edit regresses A — and adopts the one that fixes B while keeping A. Capability:

Honest scope — what's proven, what's open

the point is the boundary
■ Robust
  • The single adoption decision: accept real transfer, reject overfit / regression / lookup-hack.
  • Holds across all four of {prompt · rule · memory · tool}.
  • Holds on real code (MBPP), not just controlled tasks.
  • Embodied verifier works: a failure rate, not a single run.
▲ Open / not claimed
  • The multi-round cumulative advantage is small & noisy at scale — not a large-compounding claim.
  • Embodied fix-generation (a fix that provably lowers the rate) is unsolved — needs retention-DAgger, a multi-week build.
  • 7B model, benchmark substrates — not SOTA raw detection or production.
